# Задание 2 — API-пайплайн: данные → LLM → результат

## Цель
Написать рабочий скрипт, который автоматически отправляет данные в LLM через API и получает **структурированный ответ в JSON**, сохраняя результат в файл.

## Выбранная задача
**Классификация отзывов:** скрипт читает русскоязычные отзывы → LLM определяет:
- `sentiment`: `positive/negative/neutral`
- `topic`: тема (2–5 слов)
- `summary`: краткое резюме (1 предложение)

## Что делает пайплайн
1) Читает входные данные из `task2/input.csv` (колонки: `id`, `review`)  
2) Для каждой строки отправляет запрос в **Groq API** (LLM через API)  
3) Получает ответ LLM в виде **строго одного JSON-объекта** формата:

```json
{"sentiment":"positive/negative/neutral","topic":"...","summary":"..."}
```

4) Парсит JSON (если парсинг не удался — логирует и записывает `null`)  
5) Сохраняет результат в `task2/output.csv` (колонки: `id`, `sentiment`, `topic`, `summary`)

## Установка

Из корня проекта:

```bash
py -3 -m pip install -r task2/requirements.txt
```

## Ключ API

Ключ берётся из файла `.env` в корне проекта.

```bash
copy task2\.env.example .env  # Windows PowerShell
```

В `.env` укажите:

`GROQ_API_KEY=...`

Где взять ключ:
- Откройте `https://console.groq.com/`
- Войдите или зарегистрируйтесь в аккаунте
- В меню слева выберите **API Keys**  → **Create API key**
- Скопируйте созданный ключ и вставьте его в файл `.env`

## Запуск

```bash
py -3 task2\pipeline.py
```

Результат появится в `task2/output.csv`.
